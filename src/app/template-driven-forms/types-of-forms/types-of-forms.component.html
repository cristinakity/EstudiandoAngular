Types of forms

Tipos de formularios que podemos utilizar en Angular: Template-Driven forms y Reactive forms.

En angular tenemos el FormControl, para cada elmento input en nuestro formulario necesitaremos crear una instancia de la clase FormControl y con esto podemos revisar las siguietne propiedades:

  • value: Valor almacenado en el input.
  • touched: Si el input ha sido accedido o tocado.
  • untouched: Si el input no ha sido accedido o tocado.
  • dirty: Si el valor ha cambiado.
  • pristine: Si el valor no ha sido cambiado.
  • valid: Si cumple con las reglas como requiered, maxLength, en pocas palabras si es valido.
  • errors: Aqui se listaran los errores de validaci√≥n si no cumple con las reglas asignadas a este input, o si no es valido.

Tambine tenemos otra clase llamada FormGroup la cual representa un grupo de elementos de tipo FormControl. Todas la propiedades que estasn disponibles en FormControl tambien estan disponibles en FormGroup. Cuando preguntamos a un FormGroup si es valido o no este validara todos los FormControl dentro de el y sera valido cuando todos sean validos, De igual manera podemos obtener todos los errores de validaciones (la propiedad errors) dentro del FormGroup, esto es ma facil que revisar o iterar cada uno de los FormControl.

Entonces para gregar validaciones a nuestro formulario necesitamos crear:

  • FormGroup: 1 FormGroup para el formulario.
    • FormControl: N FormControls, uno por cada elemento input del formulario.

Hay dos formas de crear estos objectos:

  1. Directives: Aplicando algunas directivas al template y angular creara estos objetos implicitamente.
    Formularios creados de este modo se conocen como Template-Driven forms.
    Template-Driven forms: Signfica que son construidos utilizando el Template.
  2. Code: Escribiendo codigo en Typescript para crear estos objetos explicitamente. Tendremos que escribir codigo para crear nuevas instancias de FormGroup y FormControl.
    Formulario creados de esta manera se conocen como Reactive forms.

Diferencias entre Template-Driven forms y Reactive forms

Template-Driven
  • Es bueno para crear formulario sencillos o simples.
  • Se utiliza para validaciones simples o sencillas.
  • Son faciles de crear.
  • Requieren menos codigo.
Reactive
  • Tenemos mas control sobre la logica de las validaciones. Para crear validaciones muy complejas tendremos que utilizar los Reactive forms.
  • Es bueno para formularios complejos.
  • Nos permite hacer UnitTest para la logica de las validaciones.
