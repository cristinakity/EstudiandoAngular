View Encapsulation

Para explicar view encapsulation en Angular es necesario explicar antes el concepto de Shawod DOM.

Shadow DOM

Nos permite aplicar scoped styles a elementos. Es una especificacion que habilita el arbol de estilos del DOM.

Es algo relativamente nuevo y no funciona en versiones mas viejas de los browsers.

Cuando creamos estilos como por ejemplo:
h1{
    color:red;
}
Y no queremos que se progame en TODA nuestra aplicacion, osea queremos que solo el h1 que estamos definiendo en este scope de estilo tenga texto color rojo.
Si se habilita, aqui es donde el Shadow DOM hace su magia.

encapsulation

En Angular Component tenemos la propiedad encapsulation, en la cual el valor por default es emulated.
Esta propiedad nos permite definir el tipo de shadow DOM que utilizara Angular para el componente.
El valor de encapsulation es de tipo ViewEncapsulation, el cual es un enum definido en @angular/core


ViewEncapsulation

  1. Emulated: Angular crea su propio ambiente de shadow DOM emulado, le asigna tipo alias a los styles para poderlos definir como unicos.
    Este tipo de encapsulamiento es el que angular asigna por default y el recomendado para usar, ya que como no todos los browser soporta Shadow DOM con esta opcion Angular creara una emulacion del Shadow DOM permitiendonos tener esta funcionalidad en browsers que no soportan Shadow DOM.
  2. Native: Angular usara el Shadow DOM igual que en Javascript.
    Desventaja de usar este tipo es que no sera soportado en todos los browsers o en todas sus versiones.
  3. None: No se usara ningun tipo de Shadow DOM o encapsulamiento, osea que estos estilos se propagaran en toda la aplicacion.
