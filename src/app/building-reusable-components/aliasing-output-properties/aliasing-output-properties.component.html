Aliasing Output Properties

Podemos usar Alias para mantener el contrato del componente estable. Si hacemos cambios en algun output propertie y este property tienes un alias, no sera necesario cambiar el output property en el padre.

El alias se agrega como un string dentro del Decorator @Output('alias'), al momento de declara la propiedad.

Es util poner alias cuando por motivos de refactoring cambiamos los nombres de las variables. Al tener un alias el nombre queda igual en el componente padre donde estamos utilizando el evento, por ende son menos lugares donde tenemos que cambiar datos.
