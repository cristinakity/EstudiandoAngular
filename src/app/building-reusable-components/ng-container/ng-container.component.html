ngContainer

Sirve para pasar solo el contenido del selector al ng-content. Esto significa que en el HTML no se creara tambien el elemento del selector.

Los siguientes panels tienen definidos ng-content en el header y el body, el contenido de estos esta siendo asigando desde el componente padre.

De este modo:
  • En el hijo:
    <ng-content selector=".p1Heading"></ng-content>
    Se crea el ng-content con un selector el cual se define como queryStrings(. para clases, # para id, etc.).
    Al definir un selector; En el componente padre se creara un elemento que tenga el mismo queryString que el selector, de este modo ese elemento sera el contenido de nuestro ng-content.
  • En el padre:
    Solo pasamos el contenido dentro de elementos que coincidan con el selector del ng-content.
    <app-ng-container>
        <ng-container class="p1Heading">Panel 1</ng-container>
        <ng-container class="p1Body">
            <h5>Body</h5>
            <p>
                Este el el panel 1, dentro del boy se esta pasando HTML y no solo texto.
            </p>
        </ng-container>
    </app-ng-container>

<div _ngcontent-ofx-c72="" class="panel panel-default">
    <div _ngcontent-ofx-c72="" class="panel-heading">Panel 1
        <!--ng-container-->
    </div>
    <div _ngcontent-ofx-c72="" class="panel-body">
        <h5 _ngcontent-ofx-c73="">Body</h5>
        <p _ngcontent-ofx-c73=""> Este el el panel 1, dentro del boy se esta pasando HTML y no solo texto. </p>
        <!--ng-container-->
    </div>
</div>

* ng-container paso solamente el codigo dentro de el y no el propio selector como en ng-content.
