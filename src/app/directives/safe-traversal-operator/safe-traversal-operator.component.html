Safe traversal operator

Es el operador ? (signo de cierre de interrogacion). Se usa para validar si un objeto tiene datos, osea que no venga null.

Si tenemos el objeto:
persona {
  nombre: 'Juan Perez',
  edad: 30,
  direccion: null
}

Direccion representa otro objeto, pero en este caso viene null.
Al intentar utilizar string interpolation {{'\{\{ persona.direccion.calle \}\}'}} tendremos un error de compilacion porque direccion es null por lo tanto .calle no existe.

En cambio si el objeto tuviera con datos:
persona {
  nombre: 'Juan Perez',
  edad: 30,
  direccion: {
    'calle':'Avenida Siempre viva'
    'numero':'1029'
    'cp':'70324'
    'ciudad':'Mexico'
  }
}
Entonces esta instrucion {{'\{\{ persona.direccion.calle \}\}'}} no nos arrojaria ningun error.


Pero supongamos que direccion es null, hay dos forma de arreglarlo:

  1. Utilizando *ngIf: Validando lo siguiente *ngIf="persona.direccion"
  2. Utilizando el safe traversal operator ?: Lo cal es mas simple {{ persona.direccion?.calle }}
